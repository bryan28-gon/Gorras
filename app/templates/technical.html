Asincronismo, transacciones y hilos

Asincronismo

Ocurre en las rutas `async def` de FastAPI, por ejemplo al cargar productos, carrito, perfil e inventario.

Tambien ocurre cuando el backend ejecuta `await loop.run_in_executor(...)`: la ruta no bloquea el servidor mientras el hilo procesa inventario o compra.

app/main.py -> async def checkout()
Transacciones BD

Ocurren dentro de `with db.begin()` en el servicio de inventario. Si falla el stock, la compra no se confirma.

En checkout se descuentan existencias, se registra la compra y se vacia el carrito dentro de una sola transaccion.

app/inventory_service.py -> _checkout_sync()
Sincronizacion con hilos

Ocurre con `threading.Lock()`. Solo un hilo a la vez puede modificar stock o confirmar una compra.

Esto evita que dos cajas vendan la ultima gorra al mismo tiempo.

inventory_lock = threading.Lock()
Flujo de compra
  1. El usuario agrega gorras al carrito.
  2. Al comprar, FastAPI recibe la peticion de forma asincrona.
  3. La venta se envia al pool de hilos con `run_in_executor`.
  4. El hilo toma el `Lock`, abre transaccion, valida stock, descuenta inventario, crea la compra y borra el carrito.
  5. La transaccion se confirma; despues el `Lock` se libera para la siguiente venta.
Archivos principales
  • app/main.py: rutas asincronas, login, registro, catalogo, carrito, inventario y perfil.
  • app/inventory_service.py: hilos, lock, transacciones y operaciones criticas de stock.
  • app/models.py: tablas users, products, cart_items y purchases.
  • app/db.py: conexion async para web y conexion sincronica para los hilos.
